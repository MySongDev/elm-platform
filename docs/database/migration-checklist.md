# 数据库迁移检查清单

涉及 Prisma schema 或 migration 的 PR 需要完成本清单。

## 安全性

- [ ] 是否会锁大表？
- [ ] 是否会长时间阻塞读写？
- [ ] 是否需要分批回填数据？
- [ ] 是否需要先上线兼容代码，再执行迁移？
- [ ] 是否会影响线上定时任务、队列或后台脚本？

## 兼容性

- [ ] 当前已部署应用是否能兼容新 schema？
- [ ] 新应用是否能兼容迁移前 schema？
- [ ] 是否涉及字段重命名、删除或非空约束？
- [ ] 是否需要先写双字段或双读逻辑？

## 回滚

- [ ] 是否可以回滚 migration？
- [ ] 回滚是否会丢数据？
- [ ] 是否需要手工修复脚本？
- [ ] PR 是否说明了失败后的恢复步骤？

## 索引与性能

- [ ] 是否新增索引？
- [ ] 索引是否适合线上数据规模？
- [ ] 是否可能造成写入变慢？
- [ ] 是否需要先观察慢查询或执行计划？

## 数据与测试

- [ ] 是否需要更新 seed 数据？
- [ ] 是否需要补充迁移后的查询测试？
- [ ] 是否需要更新接口契约测试或前端 fixture？
- [ ] 是否需要在测试环境执行一次完整 migration？
